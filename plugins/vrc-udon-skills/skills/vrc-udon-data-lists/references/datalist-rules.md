# DataList Rules

## 基本原則

- DataList は明示指定時のみ使う
- 新規案件のデフォルトは固定長配列
- 既存資産が DataList 前提なら、その互換性を優先する

## 採用してよい場面

- ユーザーが明示的に DataList 利用を求めた
- 既存コードや既存データモデルが DataList 前提
- 可変長コンテナであること自体が要件

## 採用しない場面

- 固定長配列で十分
- 単純な状態管理で足りる
- 導入理由が曖昧

## 実装ルール

- なぜ配列ではなく DataList なのかを説明できるようにする
- 読み書き箇所を散らさない
- 他の状態表現との二重管理を避ける
- 導入先プロジェクトの保守者が読める形で残す
