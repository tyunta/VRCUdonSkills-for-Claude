# Transfer Rules

## 基本原則

- `import/` と `export/` は必要な案件でだけ使う
- 正本は常に project-owned 文書側に置く
- 外部情報は採用した内容だけ翻訳して残す

## import/

使ってよい場面:

- 外部ブログ、記事、配布資料、断片メモを一時的に受けるとき
- まだ project-owned 文書へ整理していない材料を置くとき

使い終わったら:

- 採用内容を project-owned 文書へ反映する
- 原文を正本扱いしない

## export/

使ってよい場面:

- 他案件へ再利用したい know-how を整理して出力するとき
- Skill 化前の中間成果物を配布単位でまとめるとき

置かないもの:

- 一時メモ
- プロジェクト固有すぎる内部事情
- 正式仕様を置き換える未整理メモ
